# Video Recording

Use `record` to capture browser automation as a WebM video for debugging, CI evidence, product walkthroughs, or repro reports.

## Basic workflow

```bash
agent-browser open https://example.com
agent-browser record start ./demo.webm
agent-browser snapshot -i
agent-browser click @e1

agent-browser record stop
```

After launching a session, `record start` can also navigate immediately:

```bash
agent-browser open
agent-browser record start ./demo.webm https://example.com
```

If no URL is provided, recording starts from the current page. The recording context copies cookies from the active session.

## Recording effects

The default `cursor` preset renders a synthetic cursor into the video because CDP screenshots do not include the OS cursor. The default cursor preset renders effects in the recorded page. Demo mode records a stable compositor surface so cursor, overlay, spotlight, and zoom animations remain independent of page navigation. Clicks show cursor flight plus click ripple only. Use `--record-mode demo` for slower presentation cursor timing with the cursor visible between actions, animated input, blocking visual clicks, and automatic pauses between actions so agent inference delays do not create idle video. Add zoom and overlay emphasis explicitly.

The default `--cursor-motion auto` renders the cursor while effects are active. Demo mode uses `always` unless you pass an explicit cursor motion flag.

```bash
agent-browser record start ./demo.webm --record-mode demo
agent-browser record overlay text "Choose a plan" --position bottom
agent-browser record zoom to @e4 --scale 1.45
agent-browser record zoom reset
```

## CI evidence

```bash
#!/bin/bash
set -e

cleanup() {
  agent-browser record stop 2>/dev/null || true
  agent-browser close 2>/dev/null || true
}
trap cleanup EXIT

agent-browser open https://app.example.com/login
agent-browser record start "./artifacts/login-flow.webm"
agent-browser snapshot -i
agent-browser fill @e1 "demo@example.com"
agent-browser fill @e2 "password"
agent-browser click @e3
agent-browser wait --url "**/dashboard"
```

Keep recordings as CI artifacts when browser failures are hard to diagnose from text output alone.

## Human-readable demos

Use demo mode when the recording is meant to be watched by a person rather than used as a debugging artifact. Demo capture pauses between visual actions, so separate tool calls and bare waits do not create dead air in the final video.

```bash
agent-browser open https://shop.example.com
agent-browser record start ./checkout.webm --record-mode demo
agent-browser record overlay text "Open the cart" --position bottom
agent-browser click @e4
agent-browser record zoom to @e6 --scale 1.4
agent-browser screenshot ./screenshots/cart.png
agent-browser record zoom reset
agent-browser record stop
```

Screenshots and videos work well together: screenshots capture precise still states, while the video shows timing, transitions, and unexpected overlays.

## Limitations

- Recording adds overhead to automation.
- Long recordings can use significant disk space.
- Use `record stop` before closing a session if you need the file flushed.
- Some constrained headless environments may have codec or GPU limitations.
